# WeKit CGI 超时问题修复 - 项目索引

## 📌 修复概要

**问题**：脚本在网络正常的情况下调用 `wechat.sendCgi()` 超时  
**原因**：主线程与后台线程死锁（互锁）  
**解决**：框架层自动检测线程，如果在主线程则用后台线程等待响应

## 🔍 快速参考

### 问题现象
```
Error: Timeout waiting for CGI response
```

### 快速诊断
```
在后台线程中运行脚本的CGI操作：
new java.lang.Thread(function() {
    var result = wechat.sendCgi(...);
}).start();
```

### 修复验证
```
1. 运行改进的脚本
2. 在 Logcat 搜索 "CGI" 或 "Result"
3. 应该看到 JSON 响应，而不是 "Timeout"
```
